# Development Workflow

このドキュメントには、日常の開発フロー、レビュー前の確認、ブランチ運用を記録する。

## 基本フロー

1. 作業内容に関連する `docs/` のドキュメントを確認する。
2. 必要に応じて、仕様、設計、実装の差分を明示する。
3. 小さく変更し、関連する mise task で検証を実行する。
4. 変更に合わせてドキュメントを更新する。
5. レビュー前に差分と検証結果を確認する。

## コマンド方針

- 開発コマンドは `mise tasks` に集約する。
- 可能な限り `pnpm`、`biome`、`expo` などを直接実行せず、`mise run ...` 経由で実行する。
- 新しい開発コマンドを追加する場合は、先に `mise.toml` に task として定義する。
- ツールは `mise.toml` で管理し、ローカル環境に暗黙依存しない。

## レビュー前確認

- 関連する `docs/` が更新されているかを確認する。
- 実行した mise task と結果を記録する。
- 秘密情報や不要なローカル設定が含まれていないかを確認する。
- 仕様、設計、実装の差分が残っている場合は、判断が必要な点を明示する。
