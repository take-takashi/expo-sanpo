# Development Documents

このディレクトリには、日常開発に必要な手順や判断基準を配置する。

主な対象は、ローカルセットアップ、開発コマンド、タスクランナー、テスト、Lint、Format、ブランチ運用、レビュー前の確認である。

本番運用、障害対応、監視、バックアップ、データ修正手順は `docs/operations/` に配置する。

## ファイル構成

- `setup.md`: ローカル開発環境の準備手順を記載する。
- `commands.md`: よく使う開発コマンドとタスクランナーの入口を記載する。
- `workflow.md`: 日常の開発フロー、レビュー前確認、ブランチ運用を記載する。
- `testing.md`: Unit、Component、Integration、E2E のテスト方針を記載する。
- `pnpm-workspace.md`: pnpm workspace の使い方と、このリポジトリでの構成を記載する。
