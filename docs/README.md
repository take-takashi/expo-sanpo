# Documentation

このディレクトリには、`expo-sanpo` のプロダクト、仕様、設計、開発、運用に関する文書を配置する。

## 構成

| ディレクトリまたはファイル | 役割 |
| --- | --- |
| `product/` | 何を、なぜ作るのかを整理する |
| `specs/` | 機能がどのように振る舞うべきかを整理する |
| `architecture/` | システム構成、技術方針、技術スタックを整理する |
| `development/` | ローカル開発、コマンド、日常の開発フローを整理する |
| `decisions/` | 重要な判断とその理由を記録する |
| `operations/` | リリース、障害対応、環境管理などの運用手順を整理する |
| `glossary.md` | 用語の意味や表記を整理する |
| `lessons.md` | AI 協働時に繰り返し参照すべき教訓を記録する |

## 更新方針

- 仕様、設計、実装が変わった場合は、関連する文書も同じ変更単位で更新する。
- 図を記述する場合は、原則として Markdown 内に Mermaid で記述する。
- 未確定の内容は推測で埋めず、`TBD` として残すか、判断が必要な点を明示する。
