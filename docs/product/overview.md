# Product Overview

## 概要

`expo-sanpo` は、iPhone から母艦の Mac 上で動作する Codex CLI にプロンプトを送り、散歩中でも AI と雑談したり、開発作業や調査作業を進めたりできるようにする iPhone アプリである。

アプリは React Native と Expo で実装する。

## 背景

散歩中など、Mac の前に座っていない状況でも、AI と会話したり、思いついた作業や確認したいことを Codex CLI に依頼したりしたい。

通常のターミナル操作を iPhone 上で完全に再現するのではなく、プロンプト送信と結果確認に用途を絞ることで、移動中でも扱いやすい体験を目指す。

音声読み上げが実現できる場合は、画面を見続けずに AI の応答を把握できるため、散歩中の雑談体験に近づく。一方で、音声読み上げは実装難度や品質面の不確実性があるため、初期段階では必須条件にしない。

## 主要な利用シーン

- 散歩中に思いついた実装方針や調査内容を Codex CLI に依頼する。
- 散歩中に AI と雑談し、考えごとやアイデアを広げる。
- 音声読み上げが難しい場合でも、散歩中に開発指示や調査依頼を送る。
- 母艦の Mac 側ブリッジに接続し、既存の開発環境と認証情報を利用して作業する。
- Codex CLI の出力を iPhone で確認する。
- 可能であれば、Codex CLI の結果を TTS で読み上げ、画面を見続けずに内容を把握する。

## ユースケース優先度

| 優先度 | ユースケース | 成立条件 | 備考 |
| --- | --- | --- | --- |
| 高 | 散歩中に AI と雑談する | プロンプト送信、応答確認、可能なら音声読み上げ | 読み上げがあると体験価値が大きい |
| 高 | 散歩中に開発指示を送る | プロンプト送信、応答確認 | 読み上げが無くても成立する |
| 中 | 散歩中に調査依頼を送る | プロンプト送信、応答確認 | 長い応答の扱いが課題になる |
| 中 | 応答を音声で聞く | TTS、読み上げ対象の整形 | 実装難度と品質を検証する必要がある |

## スコープ

### 対象

- iPhone アプリを作る。
- React Native と Expo を使う。
- iPhone から Mac 側ブリッジに接続する。
- Mac 側ブリッジは tmux を前提にする。
- tmux 上で Codex CLI を起動する。
- iPhone アプリから Codex CLI にプロンプトを送る。
- Codex CLI の結果を iPhone アプリで確認する。
- 可能であれば、結果を TTS で読み上げる。
- TTS が実現できない段階でも、テキストでプロンプトを送り、結果を確認できるようにする。

### 対象外

- VPN の構築や管理は対象外とする。
- VPN には Tailscale を使う前提とする。
- iPhone アプリ内に汎用ターミナルエミュレータを作ることは、現時点では主目的にしない。
