# Architecture Documents

このディレクトリには、システム全体の構成や技術方針を説明するドキュメントを配置する。

主な対象は、アーキテクチャ概要、データモデル、外部連携、技術選定、非機能要件、境界づけの整理である。

技術スタック、採用理由、主要ツール、バージョン方針は `tech-stack.md` に記載する。

初期実現性、Expo Go の制約、Development Build への移行判断は `feasibility.md` に記載する。

モノレポ構成とパッケージ責務は `repository-structure.md` に記載する。

iPhone アプリと Mac 側ブリッジの API 契約は `api-contracts.md` に記載する。

Codex CLI の app-server を使った構造化イベント取得の検証結果は `codex-app-server-validation.md` に記載する。
