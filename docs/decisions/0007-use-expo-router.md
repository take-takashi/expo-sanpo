# 0007. Expo Router を使う

## 状態

採用

## 背景

iPhone アプリでは、接続先設定、セッション一覧、プロンプト入力、会話表示、設定など複数の画面が想定される。

React Navigation を直接設定する方法もあるが、Expo Router を使うとファイル構成に沿って画面遷移を管理しやすい。

## 判断

Expo Router を使う。

## 理由

- Expo プロジェクトとの統合が自然である。
- ファイルベースルーティングにより、画面追加と画面遷移の見通しが良い。
- typed routes、deep linking、web 対応などを後から活かせる可能性がある。
- 初期 PoC でも画面構成を整理しやすい。

## 影響

- iPhone アプリは `apps/mobile/src/app` に Expo Router の画面を配置する。
- 画面以外のコンポーネント、hooks、API クライアントは app ディレクトリ外に置く。
