# 0004. Zod を API 契約と実行時検証に使う

## 状態

採用

## 背景

iPhone アプリと Mac 側ブリッジの間では、プロンプト送信、セッション状態、出力イベント、エラーなどのデータをやり取りする。

TypeScript の型だけでは、ネットワーク境界を越えたデータの実行時検証ができない。

## 判断

API 契約には Zod を使う。

共有パッケージに Zod スキーマを置き、iPhone アプリと Mac 側ブリッジの両方から参照する。TypeScript 型は Zod スキーマから推論する。

## 理由

- API 境界の入力と出力を実行時に検証できる。
- 型定義とバリデーション定義を重複させずに済む。
- tmux control mode の出力をアプリ向けイベントへ変換する際にも検証を使える。
