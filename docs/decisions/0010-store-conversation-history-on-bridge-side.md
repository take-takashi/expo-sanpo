# 0010. 会話履歴の責務を Mac 側ブリッジに置く

## 状態

採用

## 背景

このプロジェクトでは、会話や作業の実体は Mac 上の tmux セッションと Codex CLI にある。

iPhone アプリは、散歩中にプロンプトを送り、応答を確認し、必要に応じて読み上げるための操作面である。

## 判断

会話履歴の責務は Mac 側ブリッジに置く。

iPhone アプリは、表示と体験を滑らかにするための一時的なキャッシュだけを持つ。

## 理由

- セッションの実体は Codex CLI と tmux にある。
- iPhone 側に履歴の正本を持つと、tmux / Codex CLI の状態とずれる可能性がある。
- 複数端末や再接続を考えた場合、Mac 側を正としたほうが扱いやすい。
- TTS や表示のために必要な整形済みメッセージを、ブリッジ側で提供しやすい。

## 影響

- `GET /sessions/:id/messages` は Mac 側ブリッジが保持または再構成した履歴を返す。
- iPhone アプリのローカル保存は、接続先、認証トークン、TTS 設定、表示用キャッシュに限定する。
- tmux control mode や `capture-pane` から取得した出力を、ブリッジ側でメッセージ形式に変換する。

## 未解決事項

- tmux / Codex CLI の出力から、ユーザー発話とアシスタント応答をどう分離するか。
- 履歴の最大件数、保持期間、秘密情報を含む出力の扱い。
