# 0011. PoC では Mac 側ブリッジの会話履歴をメモリ保持にする

## 状態

採用

## 背景

会話履歴の正本は Mac 側ブリッジに置く方針である。

初期 PoC では、履歴の永続化、保持期間、ファイル形式、秘密情報の扱いを作り込むよりも、プロンプト送信と応答確認の体験検証を優先したい。

## 判断

PoC では、Mac 側ブリッジの会話履歴はメモリだけに保持する。

ファイル永続化やデータベース利用は、必要性が見えた段階で再検討する。

## 理由

- 実装範囲を小さくできる。
- 秘密情報を含む可能性がある出力を不用意に永続化しないで済む。
- tmux / Codex CLI セッションが実体であり、PoC ではブリッジ側履歴は表示用の整形結果として扱える。

## 影響

- Mac 側ブリッジを再起動すると、ブリッジが保持する整形済み会話履歴は失われる。
- 必要に応じて、再接続時に tmux から最新表示を再構成する。
- iPhone アプリは、ブリッジの履歴消失を前提に、表示用キャッシュを正本として扱わない。

## 再検討条件

- 過去の会話履歴をセッション単位で見返したくなった場合。
- Mac 側ブリッジの再起動後も整形済み履歴を維持したくなった場合。
- TTS 用の要約履歴や読み上げ済み状態を保持したくなった場合。
