# 0009. 初期 iPhone アプリはスタック中心の画面構成にする

## 状態

採用

## 背景

初期 PoC では、接続先を設定し、Mac 側ブリッジに接続し、セッションを作成または選択し、会話画面でプロンプトを送る流れが中心になる。

タブ構成にすると、会話、セッション、設定などへ素早く切り替えられる一方で、未接続状態やセッション未選択状態の扱いが増える。

## 判断

初期 iPhone アプリはスタック中心の画面構成にする。

タブ構成は、主要機能が増えて横断的な画面切り替えが必要になった段階で再検討する。

## 理由

- 初期体験が「接続する -> セッションを選ぶ -> 会話する」という手順型である。
- 未接続状態やセッション未選択状態の分岐を単純にできる。
- PoC の実装範囲を小さくできる。
- Expo Router のファイルベース構成と相性が良い。

## 初期ルート案

| ルート | 役割 |
| --- | --- |
| `/` | 接続状態の確認と初期導線 |
| `/connections` | 接続先設定 |
| `/sessions` | tmux / Codex セッション一覧 |
| `/sessions/[id]` | 会話画面 |
| `/settings` | TTS やアプリ設定 |

## 再検討条件

- 会話履歴、接続先、設定、テンプレートなどが独立した主要機能になった場合。
- 散歩中に設定や履歴へ素早く移動する必要が強くなった場合。
- タブにしたほうが日常利用の移動量を明確に減らせる場合。
