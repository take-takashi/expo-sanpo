# expo-sanpo

Expoで散歩しながら何か作業する仕組みを作る

## ドキュメント

プロジェクトの仕様、設計、開発手順、運用手順は `docs/` 配下で管理する。

- `docs/product/`: 目的、背景、ユーザー価値、PRD、業務フロー
- `docs/specs/`: 機能仕様、画面仕様、状態遷移、受け入れ条件
- `docs/architecture/`: システム構成、技術方針、技術スタック
- `docs/development/`: セットアップ、開発コマンド、日常の開発フロー
- `docs/decisions/`: 重要な判断とその理由
- `docs/operations/`: リリース、障害対応、環境管理

AI コーディングエージェント向けの作業ルールは `CLAUDE.md` を参照する。
